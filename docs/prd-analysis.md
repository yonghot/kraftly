# PRD Analysis — Kraftly

> PRD.md 기반 우선순위 분류 + Acceptance Criteria + 의존 관계

## 우선순위 정의

- **P0 (Must Have)**: Phase 1 완료 필수. 없으면 출시 불가
- **P1 (Should Have)**: Phase 2 핵심. 비즈니스 모델 가동 필수
- **P2 (Nice to Have)**: Phase 3~4. 성장 기능

---

## P0 — Phase 1 필수 (AI Design Prototype)

### P0-1. 랜딩 페이지 ✅

**AC**:
- [ ] Hero 섹션에 CTA → Studio 이동
- [ ] How It Works 3단계 표시
- [ ] 6개 K-Design 카테고리 그리드
- [ ] 갤러리 프리뷰 (최근 12개)
- [ ] 모바일 반응형 (640px 이하)
- [ ] 영문/한글 전환 동작

**의존**: 없음 | **상태**: ✅ 완료

### P0-2. K-Design AI Studio ✅

**AC**:
- [ ] 6개 카테고리 선택
- [ ] Jewelry Type 5종 선택 (ring, necklace, earring, bracelet, brooch)
- [ ] Material 5종 선택 (silver, gold_14k, gold_18k, rose_gold, platinum)
- [ ] 사용자 프롬프트 입력 (선택)
- [ ] Generate 버튼 → Gemini API → 이미지 1장 표시
- [ ] 로딩 UX (스켈레톤 + 프로그레스 바)
- [ ] 다크 몰입형 워크스페이스
- [ ] 플로팅 프롬프트 바

**의존**: Gemini API 키 | **상태**: ✅ 완료

### P0-3. 갤러리 ✅

**AC**:
- [ ] Masonry 그리드 레이아웃
- [ ] 카테고리 필터, 타입 필터
- [ ] 인기순/최신순 정렬
- [ ] 비로그인 열람 가능

**의존**: P0-2 | **상태**: ✅ 완료 (Mock 데이터)

### P0-4. Supabase Auth 연동 ⬜

**AC**:
- [ ] 이메일/비밀번호 회원가입
- [ ] 이메일/비밀번호 로그인
- [ ] 세션 유지 (SSR 쿠키)
- [ ] 로그아웃
- [ ] 로그인 상태에 따른 UI 분기

**의존**: Supabase 프로젝트 생성, 환경변수 설정 | **상태**: ⬜ UI만 구현

### P0-5. 디자인 DB 저장 ⬜

**AC**:
- [ ] 생성된 디자인 Supabase에 저장
- [ ] 내 디자인 목록 조회
- [ ] 갤러리 공개/비공개 전환
- [ ] 좋아요 기능 (로그인 필요)

**의존**: P0-4 (Auth) | **상태**: ⬜ API 스텁만 구현

### P0-6. Rate Limiting ⬜

**AC**:
- [ ] 비로그인: 하루 2회 생성 제한
- [ ] 로그인 무료: 하루 5회 생성 제한
- [ ] 제한 초과 시 안내 메시지

**의존**: P0-4 (Auth) | **상태**: ⬜ 미구현

### P0-7. 테스트 인프라 ✅

**AC**:
- [ ] 핵심 로직 단위 테스트
- [ ] 컴포넌트 렌더링 테스트
- [ ] API 라우트 통합 테스트
- [ ] 62개+ 테스트 통과

**의존**: 없음 | **상태**: ✅ 완료 (62 tests, 11 suites)

## P1 — Phase 2 핵심 (Artisan Matching & Order)

### P1-1. 장인 프로필 등록/관리

**AC**:
- [ ] 장인 프로필 CRUD (Supabase 연동)
- [ ] 포트폴리오 이미지 업로드
- [ ] 전문 분야, 재료, 기법 선택
- [ ] 인증 마크 (관리자 승인)

**의존**: P0-4 (Auth), Supabase Storage | **상태**: ⬜ (Mock 8명 구현)

### P1-2. 장인 매칭 알고리즘

**AC**:
- [ ] 규칙 기반 점수: specialty 40% + material 25% + rating 20% + availability 15%
- [ ] 점수 breakdown UI 표시
- [ ] 필터 (specialty, material, location)

**의존**: P1-1 | **상태**: ✅ 순수 함수 구현 완료 (Mock 데이터)

### P1-3. 주문 Flow

**AC**:
- [ ] 디자인 → 장인 선택 → 견적 요청 → 결제 → 제작 → 배송
- [ ] 주문 상태 추적 (6단계)
- [ ] 주문 번호 생성 (KR-YYYYMMDD-NNNN)

**의존**: P1-1, Stripe 결제 | **상태**: ⬜

### P1-4. Stripe 결제

**AC**:
- [ ] 수수료 12% 자동 계산
- [ ] 에스크로 패턴 (완료 후 정산)
- [ ] 환불 처리

**의존**: P1-3, Stripe 계정 | **상태**: ⬜

### P1-5. 자동 번역 채팅

**AC**:
- [ ] 소비자-장인 1:1 채팅
- [ ] Supabase Realtime 기반
- [ ] 자동 번역 (Google Translate 또는 DeepL)

**의존**: P1-3, P0-4 | **상태**: ⬜

### P1-6. 장인 대시보드

**AC**:
- [ ] 주문 관리 (수락/거절/상태 변경)
- [ ] 프로필 관리
- [ ] 수익 대시보드

**의존**: P1-1, P1-3 | **상태**: ⬜
